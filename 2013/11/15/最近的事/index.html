最近的事

自己从昨晚开始基本上看完了《head first html and css 2nd》，其实在看这本书的过程中感觉自己越是学html就月不喜欢html，那时候很纠结，不懂要选的是什么方向。

　　后来我粗略的看了一下剩下的几个章节，那时候我就开始决定了，应该不会从事web 了，感觉linux软件开发更适合我吧。所以今天我在去上语文课之前就先去了一趟图书室借了《linux命令行和shell脚本编程宝典》《linux C 一站式编程》，其实我已经在电脑上看过《linux命令行和shell脚本编程宝典》电子版了，只不过说实话，我真的不喜欢在电脑上看书，感觉有点伤眼，所以特地去借了书，今晚也强迫自己静下来看去，从7点开始我就跑去了自习室看这本书，感觉确实很舒服，比在电脑上看舒服多了，看了2个小时左右就回来了，今晚确实有收获。在自习室确实非常安静，虽然看书的时候有些地方最好动手敲一敲代码，但是我没有带上电脑，其实影响也不大，反正回到宿舍后我自己动手弄了一遍。今晚看得很快，毕竟有些东西我早已经会了，这次只是复习而已，就算是复习，也确实学到了 一些以前没有注意的东西。比如以前我用的是ps aux 命令，现在感觉用ps -efH 更好，毕竟后面一个能显示ppid。

　　反正现在我已经打算放弃web了，所以我最近也要修改了一些vim 的插件，把web 的插件换成c/c++的，但是最近我还不会马上动手，毕竟我还要看shell 编程。我打算吧《linux命令行和shell脚本编程宝典》看完后再看《linux shell cooking book》，这本书强调动手，我以前看过几章，感觉里面讲了很多《linux命令行和shell脚本编程宝典》不提到的部分。最近我也打算吧github上面的web-vim用bash shell 写一个自动安装的脚本，不过还是要等到我看完《linux命令行和shell脚本编程宝典》再说。最近还有一个计划，打算把shell 的内容看完后，自己写一个简单的脚本，可能也叫软件吧，就是方便启动conky主题的，也就是在构想这个软件的时候我才发现sed 或则是gwak 的用处。当初在假期里面看《linux命令行和shell脚本编程宝典》的时候我看到了讲这两个软件的部分，虽然书也说了他们的强大之处，但是我还是意识不到他们能做什么。而构思这个软件，我想到一个要处理的问题，就是conky在不同的桌面要进行不同的设置，例如在unity下面要设置 own_window_type override。但是我总不能用shell调用一个文本编辑器让用户自己找到那个地方进行修改，所以我就必须用别的软件在conky 的主题中找到那个选项并修改他，好像是需要sed 来帮助的。也就是这个时候我才发现这类软件的用处，所以我想到了要学习编程，真的需要写一些实用的软件，而不是用编程来做无聊的数学题。

　　这就是自己最近的计划吧，好好加油～！
